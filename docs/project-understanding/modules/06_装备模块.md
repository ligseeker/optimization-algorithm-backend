# 06_装备模块

## 一、模块定位

装备模块管理流程图中的设备资源。它不是流程结构本身，但会被节点引用，并进入导出和优化算法。

## 二、主链

```text
ApiEquipmentController
  -> EquipmentAppServiceImpl
  -> ResourceAccessServiceImpl
  -> EquipmentMapper / ProcessNodeMapper
  -> GraphVersionServiceImpl
  -> EquipmentConverter
  -> EquipmentVO
```

## 三、关键文件

### `module/equipment/service/impl/EquipmentAppServiceImpl.java`

#### 主要职责

- 设备 CRUD
- 同 graph 下设备名称唯一
- 删除设备时清理节点上的设备引用

#### 字段逐项说明

- `equipmentMapperProvider`
  设备表读写
- `processNodeMapperProvider`
  删除设备时把相关节点的 `equipmentId` 置空
- `resourceAccessService`
  统一 graph 权限
- `graphVersionService`
  写操作后提升版本

#### 主要方法

- `createEquipment`
  校验 graph 可访问和名称唯一，插入设备

- `listEquipments`
  当前 graph 下分页查询

- `updateEquipment`
  更新前再次校验重名

- `deleteEquipment`
  先清理节点上的引用，再删设备本身

#### 最容易看不懂的点

为什么删设备时不直接拒绝，而是清理节点引用？

因为当前设计选择的是“弱引用修复”，避免节点上残留无效 `equipmentId`。
