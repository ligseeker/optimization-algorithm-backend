# 05_路径模块

## 一、模块定位

路径模块管理节点之间的连线。它的核心职责是保证图结构关系成立，而不是单纯存两列外键。

## 二、主链

```text
ApiPathController
  -> PathAppServiceImpl
  -> ResourceAccessServiceImpl
  -> ProcessPathMapper / ProcessNodeMapper
  -> GraphVersionServiceImpl
  -> PathConverter
  -> PathVO
```

## 三、关键文件

### `module/path/service/impl/PathAppServiceImpl.java`

#### 字段逐项说明

- `processPathMapperProvider`
  路径本表读写
- `processNodeMapperProvider`
  校验起点/终点节点是否合法
- `resourceAccessService`
  graph 权限校验
- `graphVersionService`
  写操作后更新 graph 版本

#### 主要方法逐项说明

- `createPath`
  先校验 graph 可访问，再分别校验起点/终点节点存在且属于该 graph，然后写入路径

- `listPaths`
  查询当前 graph 下路径列表

- `getPath`
  路径详情查询

- `updatePath`
  重新校验起点/终点节点合法性，再更新路径

- `deletePath`
  删除后提升 graph 版本

- `getPathInGraph`
  路径必须属于指定 graph

- `validateNodeExists`
  避免 path 指向其他 graph 的 node

#### 这个文件最关键的理解点

路径合法性不只看 path 自身，还要看：

- 起点节点存在吗
- 终点节点存在吗
- 它们是不是属于同一个 graph
