# 07_约束条件模块

## 一、模块定位

约束条件模块负责维护算法输入中的约束记录。  
它的业务重点不在“有一条约束数据”，而在“这条约束关联的节点是否合法、类型是否合法、编码是否唯一”。

## 二、主链

```text
ApiConstraintController
  -> ConstraintAppServiceImpl
  -> ResourceAccessServiceImpl
  -> ConstraintMapper / ProcessNodeMapper
  -> GraphVersionServiceImpl
  -> ConstraintConverter
  -> ConstraintVO
```

## 三、关键文件

### `module/constraint/service/impl/ConstraintAppServiceImpl.java`

#### 字段逐项说明

- `constraintConditionMapperProvider`
  约束本表读写
- `processNodeMapperProvider`
  校验约束关联节点属于当前 graph
- `resourceAccessService`
  graph 权限
- `graphVersionService`
  约束写操作后提升 graph 版本

#### 主要方法逐项说明

- `createConstraint`
  校验 graph、`conditionCode` 唯一、节点 1/2 合法，再写库

- `listConstraints`
  支持关键字搜索和分页

- `updateConstraint`
  先做和新增同样的合法性检查，再更新

- `deleteConstraint`
  删除后提升 graph 版本

- `getConstraintInGraph`
  限制约束必须属于指定 graph

- `validateNodeExists`
  防止约束挂到其他 graph 的节点上

#### 最容易看不懂的点

这里做的节点校验，并不是算法求解时的约束合法性校验，而是平台侧的“引用完整性校验”。
