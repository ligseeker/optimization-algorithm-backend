# 00_阅读指南与文档地图

## 这套文档是干什么的

这套文档不是“重构方案文档”，而是“把重构后的项目真正看懂”的辅助说明。

它重点回答 4 类问题：

1. 某个请求到底是怎么在后端里流转的。
2. DTO、Entity、VO、算法模型之间的数据是怎么传递的。
3. 异常是从哪里抛出的，又是怎么统一返回给前端的。
4. 某个文件为什么存在、里面各个部分分别做什么。

## 怎么用这套文档

### 看模块时，先抓 3 条线

1. 请求线：Controller 怎么进，Service 怎么编排。
2. 数据线：DTO/Entity/VO/算法模型怎么转。
3. 异常线：参数错误、权限错误、业务错误、系统错误怎么往外冒。

### 看文件时，不要只看类名

重点问自己：

1. 这个文件在调用链里的位置是什么。
2. 这个文件的输入和输出是什么。
3. 这个文件是“做事”的，还是“搬运/转换/适配”的。
4. 这个文件抛出的异常最后会被谁接住。

### 老目录不要直接忽略

仓库里旧 `Controller / Service / algorithm` 还在，所以如果只看新 `module` 会出现理解断层。先看旧结构映射，再决定某个旧文件现在是否值得深入。
