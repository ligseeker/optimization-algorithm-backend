# 04_简历项目表达与亮点提炼

## 一、项目怎么一句话介绍

推荐表述：

“将原有基于 YAML/JSON 文件的流程优化算法工具，重构为基于 Spring Boot、MyBatis-Plus、MySQL、Redis、Sa-Token 的流程优化任务管理平台，实现了流程图结构化管理、YAML 导入导出、异步优化任务执行与结果追踪。”

## 二、简历里最值得写的亮点

### 1. 平台化重构

可写：

- 负责将文件驱动的流程优化工具重构为平台化后端，建立工作空间、流程图、节点、路径、设备、约束、优化任务、优化结果等结构化资源模型。

### 2. 权限与资源归属链

可写：

- 基于 Sa-Token 和资源归属链设计权限模型，实现 workspace -> graph -> 子资源 的访问控制收口。

### 3. 缓存设计

可写：

- 使用 Redis 为流程图聚合详情和优化任务状态/结果提供缓存，加速详情读取和任务状态查询，并通过 `graph_version` 控制展示态缓存一致性。

### 4. 异步任务平台化

可写：

- 基于 `ThreadPoolTaskExecutor` 实现优化任务异步执行，支持任务提交、状态流转、失败重试和结果回查。

### 5. 算法接入与模型转换

可写：

- 保留原有算法核心，通过 `ProcessMapConverter` 打通数据库实体、YAML 结构与算法模型，实现渐进式重构而非推倒重写。

### 6. 导入导出与错误报告

可写：

- 实现 YAML 导入导出能力，并对导入阶段的结构错误提供结构化错误报告，提升数据接入可用性与排错效率。

## 三、写简历时要避免的说法

不建议写成：

- “设计了分布式架构”
- “搭建了高并发调度平台”
- “实现了完整权限系统”

因为当前项目更准确的定位是：

- 单体后端中的平台化重构
- 线程池异步任务执行，而非消息队列调度平台
- 基于角色和资源归属链的权限控制，而不是复杂 RBAC 平台

## 四、推荐的项目描述长度

### 短版

适合简历项目列表：

“重构流程优化后端，将原有 YAML/JSON 文件驱动算法工具升级为基于 Spring Boot + MySQL + Redis + Sa-Token 的流程优化任务管理平台，支持流程图结构化管理、YAML 导入导出与异步优化任务执行。”

### 中版

适合简历详细项目描述：

“负责对既有流程优化算法后端进行渐进式重构，在保留原有算法核心的基础上，引入 Spring Boot、MyBatis-Plus、MySQL、Redis、Sa-Token 与线程池任务执行机制，完成工作空间、流程图及其子资源的结构化建模，实现 YAML 导入导出、优化任务异步执行、任务状态缓存、结果回查与操作日志审计能力。”
