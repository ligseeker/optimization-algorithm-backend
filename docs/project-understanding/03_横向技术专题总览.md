# 03_横向技术专题总览

## 为什么还要有专题层

如果只按业务模块写，你会反复在不同文档里看到同一类问题：

- 权限到底怎么校验
- Redis 为什么这么用
- 异步任务和事务怎么配合
- 为什么要搞 DTO / VO / Entity / Converter 这么多层

专题层的作用就是把这些横向问题集中讲透，避免每篇模块文档都重复一大段。

## 最推荐的使用方式

### 方式一：按模块看

如果你正在看 `graph` 模块：

1. 先看模块文档
2. 遇到权限问题跳到权限专题
3. 遇到缓存问题跳到 Redis 专题

### 方式二：按困惑看

如果你卡住的不是模块，而是某类问题：

- 权限 -> 权限专题
- 缓存 -> Redis 专题
- 异步任务 -> 线程池专题
- 对象转换 -> 算法接入专题
- 异常 / 事务 -> 事务与异常专题

这样你会比“到处 grep 同一个问题”快很多。
