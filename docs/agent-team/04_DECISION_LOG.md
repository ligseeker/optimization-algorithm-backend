# 决策记录

## D-001 前端技术路线采用 React + TypeScript + Vite

- 状态：已确认
- 决策：前端基础框架采用 `React 18 + TypeScript + Vite`
- 原因：这是 `AGENTS.md` 明确规定的技术栈，且适合快速搭建工程与后续模块化扩展
- 影响：后续所有前端任务必须围绕该技术路线展开

## D-002 UI 组件库采用 Ant Design

- 状态：已确认
- 决策：基础表单、表格、布局、反馈组件优先采用 `Ant Design`
- 原因：可加速后台管理型页面开发，并保证交互一致性
- 影响：页面设计以清晰、稳定、可维护为优先

## D-003 数据请求统一使用 Axios

- 状态：已确认
- 决策：所有 HTTP 调用统一收敛到 `frontend/src/api`
- 原因：符合 AGENTS 规则，便于拦截器、鉴权和错误处理统一管理
- 影响：页面和组件中不得直接编写 axios 请求

## D-004 远程数据状态统一使用 TanStack Query

- 状态：已确认
- 决策：列表、详情、轮询等服务端状态优先用 `TanStack Query`
- 原因：便于缓存、失效、重试和轮询管理
- 影响：异步轮询必须在页面卸载时停止，需基于框架能力统一处理

## D-005 客户端会话与少量本地状态使用 Zustand

- 状态：已确认
- 决策：认证态、布局态和少量跨页面本地状态使用 `Zustand`
- 原因：轻量、易拆分，适合中小型后台系统
- 影响：避免把服务端状态错误地放入本地 store

## D-006 流程图编辑器采用 React Flow

- 状态：已确认
- 决策：流程图展示与编辑基于 `React Flow`
- 原因：该库适配节点边建模场景，能够承载图编辑器能力
- 影响：后续图详情数据需要设计明确的前后端映射层

## D-007 结果可视化采用 ECharts

- 状态：已确认
- 决策：优化结果中的统计、趋势、分布图优先采用 `ECharts`
- 原因：适合数据可视化并与 React 生态兼容
- 影响：结果页需保留文本与图表双通道展示

## D-008 优先使用 `/api` 新接口，legacy 单独隔离

- 状态：已确认
- 决策：新业务优先接入 `/api` 前缀接口，旧接口仅保留兼容入口
- 原因：这是后端接口原则与 AGENTS 规则的共同要求
- 影响：legacy 代码只能存在于 `legacyAlgorithm.ts`

## D-009 接口契约先行于前端工程初始化

- 状态：已确认
- 决策：先做 Phase 0，后做 Phase 1
- 原因：当前仓库存在新旧接口并存、Sa-Token 细节待确认、OpenAPI 可用性待验证；先厘清接口可减少返工
- 影响：当前建议从 `P0-T01` 开始，而不是直接创建前端工程

## D-010 控制台文档作为 agent team 单一事实源

- 状态：已确认
- 决策：`docs/agent-team/` 作为主控 Agent 的排程、决策、问题、验证记录中心
- 原因：便于连续执行、多人协作与回溯
- 影响：每轮推进后都要同步更新相关文档

## D-011 前端接口白名单以 module 控制器为准

- 状态：已确认
- 决策：前端默认目标接口仅认 `src/main/java/**/module/**/controller/**` 暴露的 `/api/**` 资源化接口
- 原因：运行态 OpenAPI 同时暴露了大量历史控制器接口，不能直接等价为前端目标面
- 影响：`docs/frontend/API_CONTRACT.md`、后续 `frontend/src/api/**` 和 `frontend/src/types/**` 都必须以白名单为准

## D-012 顶层 Controller 包整体视为兼容层

- 状态：已确认
- 决策：`src/main/java/com/example/optimization_algorithm_backend/Controller/` 整体视为 legacy 兼容层
- 原因：该目录下接口风格与新接口体系不一致，且已被 OpenAPI 一并暴露
- 影响：除明确保留的兼容接口外，前端新业务不得默认接入该目录下接口

## D-013 前端默认通过请求头传递 satoken

- 状态：已确认
- 决策：前端默认采用请求头 `satoken: <token>` 传递登录态
- 原因：运行态 Sa-Token 日志确认 `isReadHeader=true`、`isReadCookie=true`、`isWriteHeader=false`，同时登录接口响应体返回 `token` 与 `tokenName`
- 影响：Axios 鉴权拦截器应优先实现 Header 注入；Cookie 兼容可以视为后端读取能力，而不是前端主方案

## D-014 启动校验允许使用临时端口规避现有实例冲突

- 状态：已确认
- 决策：当 `mvn spring-boot:run` 因 `8081` 已被同类实例占用而失败时，允许以临时端口做最小修复验证
- 原因：这类失败不代表应用不可启动，只代表当前默认端口已被占用
- 影响：本轮已通过 `8082` 完成补充启动验证，后续联调需避免误杀现有后端进程
