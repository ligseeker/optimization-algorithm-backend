# 开发任务清单（可追踪清单）

## Phase 3：YAML 导入导出

### 目标：旧文件数据接入新数据库

- [ ] P1 | 2h | GraphConverter 双向转换
  - `ProcessMap toProcessMap(List<Entity> entities, Long graphId)`
  - `void toEntityList(ProcessMap map, Long graphId, List<Entity>...)`
  - 注意 MultiNode.nodeList 的处理（算法合并节点列表）
  - 注意 ProcessNode.nodeID ↔ Entity.node_code 映射
  - 注意 Equipment.name 唯一性

- [ ] P1 | 2h | importfile 模块
  - importfile/controller/ImportController.java
  - importfile/service/ImportService.java + Impl
  - importfile/parser/YamlParser.java（复用 Main.readData）
  - importfile/validator/ImportValidator.java
    - 文件是否为空
    - 节点 ID 是否唯一
    - 路径起点是否存在
    - 路径终点是否存在
    - 约束关联节点是否存在
    - 约束类型是否合法
    - time/cost/precision 数值合法
  - importfile/vo/ImportResultVO.java
  - importfile/vo/ImportErrorVO.java
  - 导入失败返回具体错误报告

- [ ] P1 | 1h | 导出功能
  - importfile/controller/ExportController.java
  - importfile/service/ExportService.java + Impl
  - 从 MySQL 查询 → GraphConverter 转 ProcessMap
  - → Main.writeData 转 YAML → 文件下载
  - 优化结果导出 YAML

**Phase 3 验证标准**：
- [ ] 上传 YAML 导入成功 → 返回 graphId
- [ ] 导入不合法 YAML → 返回具体错误
- [ ] 导出 YAML 与导入内容一致
- [ ] 优化结果可导出

## Phase 4：优化任务中心

### 目标：异步优化任务中心

- [ ] P2 | 1h | optimize_task 和 optimize_result 模块
  - optimize/entity/OptimizeTask.java
  - optimize/entity/OptimizeResult.java
  - optimize/mapper/OptimizeTaskMapper.java
  - optimize/mapper/OptimizeResultMapper.java

- [ ] P2 | 2h | OptimizeConverter 转换
  - Entity 列表 → ProcessMap（供算法使用）
  - ProcessMap → optimize_result 的 JSON 字段
  - 计算 total_time / total_precision / total_cost

- [ ] P2 | 2h | AlgorithmExecutor 异步执行器
  - optimize/executor/AlgorithmExecutor.java
  - 实现 Runnable 接口
  - 线程内：查图数据 → 转 ProcessMap → 调用算法 → 保存结果
  - try-catch 包裹，失败记录 error_message
  - 每个任务创建独立 OptimizeContext

- [ ] P2 | 2h | 任务管理 Service + Controller
  - optimize/service/OptimizeTaskService.java + Impl
  - optimize/controller/OptimizeTaskController.java
  - POST /api/optimize/tasks（提交任务）
  - GET /api/optimize/tasks（任务列表）
  - GET /api/optimize/tasks/{taskId}（任务状态）
  - GET /api/optimize/tasks/{taskId}/result（任务结果）
  - POST /api/optimize/tasks/{taskId}/retry（重试）
  - 任务状态机：PENDING → RUNNING → SUCCESS / FAILED

- [ ] P2 | 1h | DiffService 简化版 Diff
  - optimize/service/DiffService.java
  - 对比 oldMap 和 newMap 的路径
  - 计算指标变化（time/cost/precision）
  - 生成 diff_json 格式

**Phase 4 验证标准**：
- [ ] 提交任务 → 返回 taskId → 状态 PENDING
- [ ] 轮询状态 → RUNNING → SUCCESS
- [ ] 结果包含 graph、diff、mapCode
- [ ] 任务失败 → 有 error_message → 可重试
- [ ] 多个任务并发执行，context 隔离
