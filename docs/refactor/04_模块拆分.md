## 2. 模块职责划分

### 2.1 common 模块

**职责**：全局基础设施。

| 组件 | 说明 |
|------|------|
| `Result<T>` | 统一返回体，包含 code/message/data |
| `PageResult<T>` | 分页返回，包含 total/page/size/records |
| `ErrorCode` | 错误码枚举 |
| `BusinessException` | 业务异常，携带错误码和错误信息 |
| `GlobalExceptionHandler` | 捕获所有异常，转换为 Result |
| `MybatisPlusConfig` | 分页插件、逻辑删除插件 |
| `RedisConfig` | Redis 序列化配置 |
| `SaTokenConfig` | Sa-Token 配置 |
| `WebMvcConfig` | CORS 跨域、拦截器链 |
| `ThreadPoolConfig` | 线程池参数配置 |

**依赖**：无内部依赖。

### 2.4 graph 模块

**职责**：流程图 CRUD、详情查询（含子资源聚合）、Redis 缓存管理。

| 层 | 说明 |
|----|------|
| Controller | `/api/workspaces/{}/graphs/*`, `/api/graphs/*` |
| Service | 流程图 CRUD、详情聚合（node/path/equipment/constraint）、缓存管理 |
| Entity | flow_graph |
| Mapper | 流程图 CRUD |
| DTO | GraphCreateRequest, GraphUpdateRequest |
| VO | GraphDetailVO（含子资源列表）、GraphListItemVO |
| Converter | `GraphConverter`: Entity ↔ ProcessMap 双向转换 |

**依赖**：workspace、node、path、equipment、constraint、common。

**缓存策略**：
- 流程图详情 Key: `graph:detail:{graphId}`
- 过期时间：30 分钟
- 失效时机：新增/修改/删除节点、路径、装备、约束时删除缓存

### 2.10 optimize 模块

**职责**：优化任务管理、异步执行、结果存储、简化版 Diff。

| 层 | 说明 |
|----|------|
| Controller | `/api/optimize/tasks/*` |
| Service | 任务提交/查询/重试、状态管理 |
| Entity | optimize_task, optimize_result |
| Mapper | 任务和结果 CRUD |
| DTO | TaskCreateRequest |
| VO | TaskVO, TaskResultVO |
| Converter | `OptimizeConverter`: Entity ↔ ProcessMap 转换 |
| Executor | `AlgorithmExecutor`: Runnable 实现，封装算法调用 |

**异步执行流程**：

```text
提交任务 → 创建 optimize_task(PENDING) → 提交线程池
    → Redis 缓存状态
    → 线程执行：状态改为 RUNNING
    → 从 MySQL 查询图数据 → Converter 转为 ProcessMap
    → 调用 Algorithm1/2/3
    → 生成结果 ProcessMap → 计算 Diff
    → 保存 optimize_result
    → 缓存结果到 Redis
    → 状态改为 SUCCESS
```

**关键注意事项**：
- 不允许在单例 Service 中保存任务上下文
- 每个任务创建自己的 OptimizeContext 对象
- 算法执行失败不能导致服务崩溃
- 线程池参数可配置

**依赖**：graph（查询图数据）、node、path、equipment、constraint、common、algorithm（核心算法）、algorithm.model（复用 ProcessMap）。

### 2.12 algorithm 包（保持不动）

**职责**：核心优化算法，尽量少改。

| 组件 | 说明 |
|------|------|
| Algorithm1 | 潜力导向优化 |
| Algorithm2 | 约束强制满足优化 |
| Algorithm3 | 层次化禁忌搜索 |
| ProcessMap | 算法核心输入/输出模型 |
| MultiNode | 算法内存节点（含合并子节点） |
| ProcessPath | 算法路径模型 |
| ConstraintCondition | 算法约束模型 |
| Equipment | 算法装备模型 |
| InputInfo | 算法输入聚合 |
| utils | 节点合并/拆分/恢复等功能 |
| Constant | 约束类型枚举 |

**不需要修改的内容**：
- Algorithm1/2/3 的核心优化逻辑
- ProcessMap、MultiNode、ProcessPath、ConstraintCondition、Equipment 的字段定义
- utils 中的节点合并/拆分逻辑

**需要提供新接口的类**：
- utils 中可以新增 `fromEntityList(...)` / `toEntityList(...)` 类型转换方法（或放入 GraphConverter）

## 3. 模块间依赖关系

```text
auth ──→ common
workspace ──→ auth, common
graph ──→ workspace, common
node ──→ graph, common
path ──→ graph, common
equipment ──→ graph, common
constraint ──→ graph, common
importfile ──→ graph, node, path, equipment, constraint, common, algorithm.model
optimize ──→ graph, node, path, equipment, constraint, common, algorithm.*, algorithm.model
log ──→ auth, common
```

无循环依赖。
